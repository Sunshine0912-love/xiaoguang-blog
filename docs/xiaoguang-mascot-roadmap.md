# 小光老师虚拟形象演进路线图

> 当前版本：Phase 1 — 原图 PNG + CSS 容器级轻动态
> 主形象文件：`source/images/xiaoguang-teacher-main.png`
> 最后更新：2026-06-06

---

## 概述

小光老师的博客虚拟形象是一只叼着胡萝卜的卡通萨摩耶。形象演进分为三个阶段，从纯 CSS 容器动画逐步升级到可交互的 Live2D/Rive 虚拟助手。

每个阶段的升级判断标准：当前阶段的使用反馈 + 技术验证 + 博客整体定位是否需要更高保真度的形象。

---

## Phase 1：静态 PNG + CSS 轻动态 ✅ 当前

**已实现。**

当前主图必须使用主人确认的附件原图：1254×1254 PNG，内容为室内背景下的小光萨摩耶叼着巨型胡萝卜。入口按钮应完整显示这张图的主体，不再使用横版替代图或手绘 SVG 作为主展示形象。

### 能力

| 动画 | 效果 | 触发条件 |
|------|------|----------|
| 呼吸浮动 | 上下 7px 浮动 | 待机时自动 |
| 微缩放 | 5% 缩放呼吸 | 待机时自动 |
| 光晕脉冲 | 橙色光环呼吸 | 待机时自动 |
| 星光环 | 虚线旋转淡入淡出 | hover 时 |
| 弹跳入场 | 弹性缩放序列 | 点击打开面板 |
| 思考脉冲 | 快速缩放 + 蓝青光晕 | 流式生成回答时 |
| 思考星光 | 快速旋转光环 | 流式生成回答时 |
| 开心弹跳 | 缩放回弹 | 回答完成后 |

### 技术栈

- 单张 PNG 图片（1254×1254 RGB）
- CSS `@keyframes`：`xg-float`、`xg-mascot-breathe`、`xg-glow-pulse`、`xg-sparkle-ring`、`xg-bounce-in`、`xg-think-pulse`、`xg-think-glow`、`xg-think-sparkle`、`xg-happy-bounce`
- `::before` / `::after` 伪元素做光环
- 无 JS 动画依赖，纯 CSS 驱动
- 图片路径必须通过 Hexo `url_for('/images/xiaoguang-teacher-main.png')` 或脚本路径推导，不能写死为 `/images/...`，因为线上博客部署在 `/xiaoguang-blog/` 子路径下。

### 局限性

- 无法分层动画（眼睛、耳朵、胡萝卜、尾巴不能独立动）
- 没有表情切换
- 没有视线跟随
- 圆形裁切会损失部分画面边缘

### 升级判断条件

- 博客访问量显著增长，需要更有「活」感的形象
- 读者反馈希望小光形象更生动
- 有富余时间做图像拆层工作

## Phase 2：分层 PNG + CSS 逐部件动画 📋 计划中

### 目标

在保留 PNG 高画质的前提下，通过拆层实现部件级动画，让萨摩耶「活」起来。

### 需要准备

1. **拆层素材**（需要设计师或 AI 辅助）：
   - 身体/头部（底层）
   - 左耳 / 右耳（独立旋转）
   - 左眼 / 右眼（含瞳孔，可做眨眼/视线跟随）
   - 嘴巴/舌头
   - 胡萝卜 + 胡萝卜叶
   - 尾巴
   - 蓝领结/徽章
   - 爱心/星星装饰

2. **CSS 实现**：
   - 每个部件独立 `<div>` 或层叠 `<img>`
   - 分层 `@keyframes`：眨眼（3-5s 间隔）、耳朵轻颤、尾巴摇摆、胡萝卜微晃
   - 视线跟随鼠标（JS `mousemove` → CSS `transform`）
   - 点击时弹跳/害羞表情切换

### 技术栈增量

- 8-12 层透明 PNG（每层约 20-50KB，总计 < 300KB）
- CSS `position: absolute` 层叠
- JS `mousemove` → 视线跟随
- 可选：CSS `clip-path` 做眨眼

### 动画清单

| 动画 | 说明 |
|------|------|
| 待机呼吸 | 全身微缩放 |
| 眨眼 | 3-6s 间隔，快速闭眼-睁眼 |
| 耳朵轻摆 | 左右耳交替微旋 |
| 尾巴摇摆 | 底部左右摆 |
| 胡萝卜微晃 | 随呼吸微动 |
| 视线跟随 | 瞳孔跟随鼠标 |
| 点击弹跳 | 全身弹跳 + 开心表情 |
| 思考挠头 | 思考状态专用 |

### 升级判断条件

- 拆层素材准备完成
- 验证 CSS 层叠性能（移动端 60fps）
- 确认不增加首屏 LCP

## Phase 3：Live2D / Rive / Spine 全动态虚拟形象 🔮 远期

### 目标

把萨摩耶做成一个完整的虚拟助手形象，支持多表情、多动作、语音同步、触摸交互。

### 技术选型

| 方案 | 优势 | 劣势 | 适合场景 |
|------|------|------|----------|
| **Rive** | 矢量、体积小、有状态机、Web 运行时成熟 | 需要 Rive 编辑器 | Web/App 通用 |
| **Live2D Cubism** | 行业标准、生态完善、SDK 成熟 | 授权费用、大文件 | VTuber/虚拟主播 |
| **Spine** | 骨骼动画、游戏行业标准 | 运行时较重 | 游戏/互动应用 |
| **CSS + SVG 重绘** | 零依赖、体积最小 | 能力有限 | 纯静态博客 |

**推荐 Evolution 路径：Rive → 如果需要更复杂的骨骼 → Spine**

### Rive 方案细节

1. 用 Rive 编辑器绘制/导入萨摩耶矢量形象
2. 设置骨骼和网格变形
3. 创建状态机：
   - `Idle` → 待机呼吸 + 偶尔眨眼 + 耳朵动
   - `Hover` → 看向鼠标 + 微笑
   - `Talking` → 嘴型同步 + 头微点
   - `Thinking` → 歪头 + 眼睛转 + 胡萝卜晃
   - `Happy` → 跳起 + 爱心冒出
   - `Sleepy` → 闭眼 + 慢呼吸（夜间模式）

### 集成方式

```html
<canvas id="xiaoguang-rive" width="200" height="200"></canvas>
<script src="https://unpkg.com/@rive-app/canvas@latest"></script>
<script>
  const rive = new rive.Rive({
    src: '/rive/xiaoguang-teacher.riv',
    canvas: document.getElementById('xiaoguang-rive'),
    autoplay: true,
    stateMachines: 'Main',
  });
</script>
```

### 与博客助手集成

- 面板打开 → 触发 `Active` 状态
- 用户输入 → 触发 `Listening` 状态
- 流式输出 → 触发 `Talking` 状态（嘴型随字数）
- 回答完成 → 触发 `Happy` 状态
- 夜间 → 触发 `Sleepy` 状态

### 升级判断条件

- 博客活跃用户显著增长，有互动需求
- 有设计师/时间做 Rive 建模
- Phase 2 运行稳定至少 3 个月
- 移动端性能测试通过
